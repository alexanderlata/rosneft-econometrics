# Установка ПО для Анализа данных в Python (Эконометрика)

Наиболее оптимальный способ установить на рабочий компьютер Python с необходимым окружением – воспользоваться менеджером пакетов **Anaconda**. Помимо собственно интерпретатора языка Python, она содержит систему управления библиотеками `conda` (установка, удаление, обновление библиотек и проч.). ПО работает на всех основных операционных системах: Windows, MacOS, Linux.

Мы будем использовать минимальную установку Anaconda — **Miniconda**, а затем вручную установим только те библиотеки, которые нам будут необходимы (pandas, statsmodels и др.). Тем самым мы сэкономим место на жёстком диске и научимся контролировать своё окружение.

Для установки необходимо зайти по [ссылке (сайт conda)](https://www.anaconda.com/docs/getting-started/miniconda/main) и скачать актуальную версию для своей операционной системы.  
*(Для MacOS, как правило, удобнее скачивать файл с расширением `.pkg`, а для Windows — `.exe` установщик).*

Базово работа с Miniconda основана на работе с командной строкой. Этого не нужно бояться, нам понадобится всего-навсего несколько базовых команд.

Пожалуйста, попробуйте самостоятельно установить Miniconda (при установке используйте предлагаемые настройки по умолчанию) и обновите пакеты до последних версий по следующей инструкции:

* **Windows**: в меню «Пуск» после установки появится папка *Miniconda3*. Зайдите туда и запустите **Anaconda Prompt (Miniconda3)**. Откроется окно с командной строкой. Введите команду `conda update --all` (обратите внимание на двойное тире перед *all*!) и нажмите Enter. На предлагаемый вопрос (Proceed? [y/n]) нужно ответить `y` [yes]. После успешного обновления выполните команду `exit`, чтобы выйти из командной строки.

* **MacOS**: запустите Терминал напрямую (Launchpad -> Другие -> Терминал или через Spotlight). В Терминале введите команду `conda update --all` (с двойным тире!) и нажмите Enter. На предлагаемый вопрос ответьте `y` [yes]. После завершения обновления выполните команду `exit`. После этого окно Терминала можно закрыть (Cmd + Q).

# Настройка виртуального окружения

В Python принято работать в изолированных окружениях: это позволяет для каждого проекта иметь свой точный набор библиотек без риска конфликтов версий. Мы создадим окружение с именем `courses`.

В командной строке (`Anaconda Prompt` в Windows, `Terminal` в MacOS) нужно выполнить следующие шаги. **Это нужно сделать только один раз!**

1. Создаём новое окружение с именем `courses` и устанавливаем туда Python:
```bash
conda create -n courses python=3.14
```
(На вопрос о подтверждении [y/n] отвечаем `y`)

2. Активируем созданное окружение. **Эту команду нужно будет выполнять каждый раз при открытии командной строки перед началом работы!**
```bash
conda activate courses
```
*(Вы заметите, что слева в командной строке вместо `(base)` появится надпись `(courses)` — это означает, что вы внутри окружения).*

# Установка основных библиотек

Убедитесь, что ваше окружение активировано (`conda activate courses`), и выполните следующие команды установки пакетов. 

Сначала установим базовые библиотеки для работы с данными, графиками, а также пакет `ipykernel`, который нужен для связи окружения с Jupyter Notebook:
```bash
conda install -c conda-forge pandas numpy scipy openpyxl jupyter jupyterlab matplotlib seaborn plotly scikit-learn ipykernel
```

Затем установим ключевые библиотеки для нашего курса Эконометрики:
```bash
conda install -c conda-forge statsmodels linearmodels
```

Если нам понадобятся специализированные Python-пакеты (например, `EconML` и датасеты Вудриджа). Их можно поставить через `pip`:
```bash
pip install econml wooldridge
```

Альтернативно в Windows для **всех** пакетов можно использовать `pip install`, однако мы рекомендуем использовать `conda` (через канал `conda-forge`) там, где это возможно, чтобы избежать конфликтов версий и проблем с компиляцией пакетов, содержащих C/C++ код.

# Добавление окружения как нового Ядра (Kernel) в Jupyter

Чтобы в запущенном Jupyter Notebook (или JupyterLab) вы могли выбрать именно ваше настроенное окружение со всеми пакетами, нам нужно "зарегистрировать" его как ядро (kernel). 

При активированном окружении (`conda activate courses`) выполните команду:
```bash
python -m ipykernel install --user --name=courses --display-name "Python (courses)"
```
Теперь, когда вы будете создавать новый ноутбук в Jupyter, в списке доступных ядер (в правом верхнем углу) вы увидите пункт **Python (courses)**. Выбирайте его для выполнения домашней работы.

# Обновление установленных библиотек (в будущем)

Если спустя время вам потребуется обновить все библиотеки до последних версий, то:
В командной строке (`Anaconda Prompt` / `Terminal`) сначала активируйте окружение, а затем выполните команду обновления:
```bash
conda activate courses
conda update --all
```
